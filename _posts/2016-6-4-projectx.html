--- layout: post title: ProjectX - Let Your Manufacturing Dreams Take Wing ---

ProjectX is a concept system developed by me and my friend Tanay Gahlot as a Term Project during B.Tech degree, that aims at automating rapid prototyping process from the perspective of a maker. The system takes in the CAD model from the user and instructs him/her on how to manufacture the model on a 3-Axis CNC. In this post I will talk all about ProjectX.

The project is inspired from the idea of Personal Fabrication, i.e. to equip a designer with tools which enables him/her to manufacture his design without consulting an expert. The system stands on a philosophy that any person with a design file and access to 3-Axis CNC machine should be able to prototype whatever he likes, irrespective of his knowledge and exposure to manufacturing.


Currently a maker in need of prototyping a design consults an expert engineer who understands the geometry of the CAD design and using CAM software packages generates instructions for the 3-Axis CNC to cut the surface of workpiece. ProjectX aims at reducing this plight of the makers by automating this entire toolpath generation process by mimicking the workflow of the expert using paradigms in the field of artificial intelligence.


ProjectX generates both, the code to be executed on the workpiece using CNC, and instructions for the user to use this code, making the process of manufacturing as simple as 3 step process for the maker. First, user uploads the design file (in the form of STL file) and details about the tool bit into the ProjectX system. Second, the system generates the machine code and instructions for the maker. Finally, the maker follows the instructions to prototype his design.


The working of ProjectX is simple. First the system generates sequence of orientations. These orientations are basically way of fixing a cuboidal workpiece on the work area based on which face of the cuboid will be on the top. For each orientation in the sequence of orientations, system generates sequence of operations, which resemble higher order operations like pocketing, drilling, roughing etc. Finally for each of these operations tool movement is determined and corresponding code is wriiten. The entire flow after generation of sequence of orientations is parallelized for quick response time. At the end of processing, the code is made available for download for the user, and the sequence of orientations is interpreted to generate instructions for the user.


A paper based on this work has been accepted at IEEE International Conference of Industrial Technology (ICIT), 2016 at Taipei. Currently in beta stage, we have experimentally hosted the system on the web for easy accessibility and bug testing. The source code of the system is completely open source and is hosted on Github. You can also visit the working prototype of ProjectX in beta phase.
